Overview

Akatabo is a full-featured business management application built on Laravel, Livewire, and Flux UI. It provides a complete workflow for managing multiple businesses, customers, inventory (fabrics, products, services, office rentals), creating quotations and invoices with live previews, recording payments, handling customer fabric requests, and generating reports — all wrapped in a clean, responsive interface with light/dark themes.

Features

Multi-Business Tenancy

Users can create and manage multiple businesses from a single account. A header dropdown (on both desktop and mobile) lets you switch between businesses instantly — all data (customers, inventory, invoices, etc.) is scoped per business via the activeBusinessId() helper. The business_user pivot table tracks membership with per-user roles.

Business Onboarding

4-step wizard: Business info → Address → Logo → Default document notes. Creates the Business record and auto-assigns the user as owner. The ?add=1 URL parameter allows existing users to create additional businesses.

Customer Management

CRUD with name, email, address. Search, sort, paginated listing. PDF export of customer directory.

Inventory

Three tabs under one page:

  • Fabrics — Roll code, color, supplier, claimed/verified/used meters, pricing, image. Computed remaining_meters with low-stock warnings (quantity + low_stock_alert).
  • Products & Services — Name, SKU, description, buying/selling price, unit, image, quantity tracking.
  • Office Rentals — Name, location, monthly rent, description, image.

Quotations

Create/edit with live PDF-style preview. Select customer, dates, line items from inventory or custom entry. Discount (% or fixed) and tax. Auto-numbering (QOT-XXXX). Status workflow: draft → sent → accepted / converted / rejected. Convert to invoice. PDF download with QR code. Fulfilled quantity tracking per line item.

Invoices

Same structure as quotations plus: payment recording, receipt numbers (RCT-XXXX), balance tracking (paid_amount vs total). Statuses: draft, sent, paid, overdue, cancelled, partial. Recurring invoice support with configurable frequency and next-run date. PDF download with receipts summary.

Payments / Receipts

List all payments with search. Methods: cash, bank, mobile_money, card, other. CSV export. Soft-deletes. Activity logged on creation and deletion.

Customer Requests (Fabric Quotations)

Public-facing forms for customers to submit fabric quotation requests. Admin reviews requests and responds with pricing/availability. Status workflow: pending → reviewed → responded.

Activity Logs

Track all significant actions across the business: logins, CRUD operations on invoices/quotations/payments/customers/inventory, and user changes. Filterable by event type and date range. Accessible to admin users only.

Reports

Period filter (30d / 90d / 12m / all). Revenue, invoiced, outstanding, customer count. Payment method breakdown. Top invoices. Recent payments. PDF export.

User Management

Admin creates/manages users within the business. Roles: admin (full access) and employee (view-only for most modules). Self-deletion prevented.

Settings

  • Profile — Name, email, resend verification
  • Business — Name, email, address, logo, default invoice/quotes/receipt notes
  • Security — Password change, 2FA enable/disable/recovery codes, passkey management
  • Appearance — Light / Dark / System theme
  • Backup — Download database backups; configurable auto-backup schedule

Multi-Business Tenancy

Akatabo supports a lightweight multi-tenancy model where a single user can belong to multiple businesses.

How it works

  • Pivot table business_user — links users to businesses with a role column (admin/employee)
  • User.business_id — the user's "home" business (set during onboarding)
  • Session active_business_id — the currently active business (set on login and on switch)
  • Helpers — activeBusiness(): ?Business and activeBusinessId(): ?int used globally across all queries

Switching businesses

A dropdown in the header (both desktop and mobile) lists all businesses the user belongs to. Clicking one sends a POST to /business/switch/{business} which updates the session. The sidebar brand name and all scoped data update accordingly.

Adding a business

From the switch dropdown, click "Add Business" which opens the onboarding wizard with ?add=1. The new business is created, the user is attached via the pivot table, and the session switches to it.

Security Features

  • Rate limiting: login (5/min per email+IP), 2FA (5/min), passkeys (10/min)
  • Production password rules: min 12 chars, mixed case, numbers, symbols, uncompromised
  • 2FA confirmation flow (TOTP)
  • Passkey (WebAuthn) passwordless login
  • Password confirmation for sensitive actions
  • Email verification

PDF Generation

PDF templates are located in resources/views/pdf/:

  • invoice.blade.php — Full invoice with logo, items, totals, receipts summary, QR code
  • quotation.blade.php — Full quotation with logo, items, totals, QR code
  • reports.blade.php — Period-based report with stats and charts
  • customers.blade.php — Customer directory listing
  • invoices-list.blade.php — Invoice listing PDF
  • quotations-list.blade.php — Quotation listing PDF

All PDFs use DejaVu Sans font, inline CSS (DomPDF does not support Tailwind), and the same layout as the on-screen preview panels.

Setup

# Install dependencies
composer install
npm install

# Environment
cp .env.example .env
php artisan key:generate
# Edit .env with your database credentials

# Migrate
php artisan migrate

# Build & serve
npm run build
php artisan serve

Queue: php artisan queue:work (database driver)
Recurring Invoices: php artisan invoices:process-recurring (schedule via cron)
Theme: Toggle light/dark/system in Appearance settings
